# 声读首页 & 页面优化方案 — 产品经理视角

---

## 一、当前首页的核心问题

从截图看，首页从上到下排列了：

```
声读 Header
├── 快速体验（输入框 + 情绪标签 + 音色选择 + 试听按钮）
├── AI 工具箱（文字配音 / 情感朗读 / AI 双播 / AI 音乐）
├── AI 声音工坊 NEW
├── AI 创作工坊
├── 精选作品（"即将上线" 空状态）
└── TabBar（AI工坊 / 灵感 / 工作台 / 我的）
```

**问题诊断**：

| 问题 | 影响 |
|------|------|
| ❌ **入口太多，用户懵了** — 一个首页 6 个功能入口，用户不知道先点哪个 | 跳出率高 |
| ❌ **"快速体验"和"文字配音"功能重复** — 都是输入文字→合成语音 | 认知混乱 |
| ❌ **"快速体验"和"AI 声音工坊"也重复** — 用户不知道区别 | 选择困难 |
| ❌ **精选作品永远是空的** — "即将上线" 四个字严重拉低产品档次 | 显得半成品 |
| ❌ **AI 声音工坊藏在中间** — 你最强的功能反而不是 C 位 | 技术亮点被埋没 |
| ❌ **"Tool Calling 驱动"用户看不懂** — 技术术语出现在 C 端界面 | 不专业 |

---

## 二、首页优化方案

### 核心思路：**从"工具箱"变成"一句话出声音"**

```
优化后首页（从上到下）：
声读 Header
├── 🎯 Hero 区 — AI 声音制作人（占首屏 60%，核心入口）
├── 🎨 场景入口（3-4 个核心场景卡片）
├── 🔥 推荐收听（精选内容卡片）
└── TabBar
```

### 具体改动：

#### 1. Hero 区 — AI 声音制作人（原"快速体验"升级）

**之前**：一个输入框 + 情绪标签 + vivi 音色卡 + 试听按钮 → 合成 50 字短音频

**之后**：升级为 AI 声音工坊的「快捷入口」

```
┌─────────────────────────────────┐
│  🎙️ 嘿，想做什么声音内容？        │
│                                 │
│  ┌─────────────────────────┐    │
│  │ 帮我录一段深夜电台...     │    │ ← 点击跳转 /workshop
│  └─────────────────────────┘    │
│                                 │
│  💡 试试说：                     │
│  [深夜电台] [情感祝福] [短视频配音]│ ← 点击直接带场景跳转
│                                 │
│  或者 → 直接输入文字快速试听 🔊   │ ← 展开简化版合成
└─────────────────────────────────┘
```

**为什么**：让用户第一秒就知道"这个产品能用一句话帮我做声音内容"，而不是面对一堆工具不知所措。

#### 2. 场景入口 — 砍到 3-4 个核心场景

**之前**：AI 工具箱有 4 个功能卡片 + 2 个工坊入口 = 6 个入口

**之后**：只保留 3-4 个核心场景

| 保留 | 改名 | 理由 |
|------|------|------|
| ✅ 情感朗读 | **情感调音台** | 这是你独有的技术壁垒 |
| ✅ AI 创作工坊 | **AI 写作 + 配音** | 创作全链路是核心用户旅程 |
| ✅ AI 音乐 | **AI 音乐** | 差异化功能，面试加分 |
| ❌ 文字配音 | 合并到 Hero 区 | 和快速体验重复 |
| ❌ AI 双播 | 从首页去掉 | 功能不够成熟，放工作台里 |
| ❌ AI 声音工坊卡片 | 合并到 Hero 区 | 升级为首屏 Hero |

#### 3. 推荐收听 — 替代空的"精选作品"

**之前**：永远显示"精选作品即将上线" 🚫

**之后**：两个选择
- **方案 A**：如果数据库有作品 → 展示真实的热门作品卡片
- **方案 B**：如果没有作品 → 展示 3-4 个**预置精品 Demo**（你自己用 AI 合成的样例，存到数据库里），让用户能直接试听

**为什么**：声音产品必须让用户先"听到"好声音，才会产生创作欲望。空状态是大忌。

## 四、TabBar 优化

### 当前 TabBar
```
AI工坊 | 灵感 | 工作台 | 我的
```

### 建议调整
```
首页 | 发现 | 创作 | 我的
```

| 位置 | 现在 | 建议 | 内容 |
|------|------|------|------|
| Tab 1 | AI工坊 | **首页** | Hero + 场景入口 + 推荐 |
| Tab 2 | 灵感 | **发现** | Discover.vue（别人的作品） |
| Tab 3 | 工作台 | **创作** | Studio.vue（我的创作） |
| Tab 4 | 我的 | **我的** | Profile + 创作库 + VIP + 设置 |

**为什么改名**：
- "AI工坊"用户不理解 → "首页"最直觉
- "灵感"太抽象 → "发现"是所有内容平台的标准用词
- "工作台"太技术 → "创作"更贴近用户心理

---

## 五、一句话总结

> **砍掉重复入口，让 AI 声音工坊占 C 位，消灭空状态，把技术术语换成场景语言。**

### 改动优先级

| 优先级 | 改动 | 工作量 |
|--------|------|--------|
| P0 | 删掉"精选作品即将上线"（换成 Demo 或隐藏） | 10 分钟 |
| P0 | 去掉"Tool Calling 驱动"文案 | 1 分钟 |
| P1 | 首页砍掉"文字配音"入口，合并到 Hero | 30 分钟 |
| P1 | TabBar 改名 | 5 分钟 |
| P1 | 路由去掉 Drama / Novel 页面 | 5 分钟 |
| P2 | Hero 区改版（AI 声音制作人入口） | 2 小时 |
| P2 | 预置 3-4 个精品 Demo 到推荐收听 | 1 小时 |
