## 三、生命周期锁定

### 状态机设计

```
📝 创作设定（模式/角色/题材/灵感）
    ↓ 点击「一键生成」
🎬 生成剧本 → 设定区永久隐藏
    ↓
📖 查看全文（只读，自动展开）
    ↓
🎭 解析角色 → 选择音色 → 配音合成
    ↓ 配音完成后
🔒 音色/情感锁定（🔒图标，不可修改）
    ↓
▶ 播放 → 🚀 发布到发现页
```

### 锁定实现

| 锁定点 | 实现方式 |
|--------|----------|
| 生成后隐藏设定 | `<template v-if="!dramaFinished">` 包裹 sections 1-4 |
| 移除再来一版 | 删除「再来一版」按钮 |
| 配音后锁音色 | `:disabled="!!activeSection?.audioUrl"` + 🔒 icon |
| 配音后隐情感 | `v-if="!activeSection?.audioUrl"` 包裹 emotion tags |
| 配音后隐合成 | `v-if="!activeSection?.audioUrl"` 包裹 synthesize button |
| 已完成徽章 | header 显示 ✅ 已完成 |

## 四、TTS v2 重试机制

### 问题

短剧分角色配音连续发送几十个 TTS 请求，火山引擎限流导致后面的请求返回空音频（`SESSION_FINISHED` 正常但无 `AUDIO_ONLY_SERVER` 帧）。

### 解决方案

```java
// TtsV2Service.java — 三层防御

// 1. 请求间隔节流（800ms 最小间隔）
private static volatile long lastRequestTime = 0;
private static final long MIN_INTERVAL_MS = 800;

// 2. 指数退避重试（最多 3 次：1.5s → 3s → 6s）
for (int attempt = 1; attempt <= MAX_RETRIES; attempt++) {
    try {
        return doSynthesize(request);
    } catch (RuntimeException e) {
        if (e.getMessage().contains("未能从火山引擎")) {
            long delay = BASE_DELAY_MS * (1L << (attempt - 1));
            Thread.sleep(delay);
        }
    }
}
```

### 面试话术

> "批量合成时火山引擎会限流，返回正常协议但空音频。我实现了三层防御：**请求间隔节流**（800ms 最小间隔避免触发限流）→ **指数退避重试**（1.5s/3s/6s 随重试次数翻倍）→ **日志告警**（warn 级别方便排查）。这是典型的**弹性设计**思路。"

## 五、播放页 UX 优化

### 生成中（流式输出）

- 只显示最后 20 行流式输出（`.slice(-20)`，避免大段文字卡顿）
- 紧凑对话格式

### 生成完成后

| 组件 | 功能 |
|------|------|
| 渐变彩条 | 顶部 purple → pink → orange 渐变装饰 |
| 统计卡 | 字数 / 角色数 / 台词句数 |
| 角色速览条 | 横滑显示所有角色 + 已分配音色名 |
| 格式化对话 | 自动展开，角色稳定色彩（同一角色始终同一颜色） |

### 角色稳定色彩映射

```javascript
const charColorPalette = ['#93c5fd', '#f9a8d4', '#c4b5fd', '#67e8f9', ...]
const charColorCache = {}
const getCharColor = (name) => {
  if (charColorCache[name]) return charColorCache[name]
  const idx = Object.keys(charColorCache).length % charColorPalette.length
  charColorCache[name] = charColorPalette[idx]
  return charColorCache[name]
}
```

## 六、数据持久化 & 重进恢复

### 问题

`dramaStreamContent` 只在流式生成时填充，刷新/重新进入时为空，导致内容区不显示。

### 修复

在 `onMounted` 恢复逻辑中：

```javascript
if (project.value?.typeCode === 'drama' && sections.value.length > 0) {
  dramaFinished.value = true
  activeSection.value = sections.value[0]
  // 恢复内容
  if (activeSection.value?.content) {
    dramaStreamContent.value = activeSection.value.content
    dramaShowFull.value = true
    // 自动解析角色（静默）
    const data = await studioApi.parseScript(activeSection.value.id)
    dramaCharacters.value = data.characters || []
    dramaLines.value = data.lines || []
  }
}
```
