# [2026-03-17] — Agent SSE 流式输出 + TTS 并行合成 + 架构修复

## 🔥 性能优化

### Agent Chat SSE 流式输出

| 文件 | 改动 |
|------|------|
| `StreamingSmartAssistant.java` | **新增** 流式 Agent 接口（返回 `TokenStream`） |
| `AgentController.java` | 新增 `POST /api/agent/chat-stream` SSE 端点，使用 `OpenAiStreamingChatModel` |
| `AiWorkshop.vue` | `sendMessage()` 改用 `fetch ReadableStream` 逐 token 显示，保留 sync fallback |

**效果**: 首 token 延迟 < 1s（vs 之前 20s+ 同步阻塞），文字实时打字效果

### Agent 简单问候快速通道

| 文件 | 改动 |
|------|------|
| `AgentController.java` | `isSimpleGreeting()` 检测简单问候（你好/嗨/hi），直接返回预设回复跳过 LLM |

**效果**: "你好" 响应 ~50ms（vs 20s+）

### Agent LLM 参数优化

| 文件 | 改动 |
|------|------|
| `AgentController.java` | `maxTokens` 1024→512，`maxMessages` 20→10 |

**效果**: 减少 LLM 输入 token 数，所有 Agent 调用加速

### TTS 并行合成

| 文件 | 改动 |
|------|------|
| `StudioWorkbench.vue` | `synthesizeSection`/`synthesizeAll` 从串行改为并发≤3，保持顺序拼接 |

**效果**: 5 片合成 ~50s → ~20s

### Agent 重置对话后记忆仍存在

| 文件 | 改动 |
|------|------|
| `AgentController.java` | `reset()` memoryId 从 `token.hashCode()` 修正为 `user.getId().toString()` |
