## 一、优化概览

本次对情感电台模块进行了**提示词工程重构 + TTS 合成加速 + 前端 UX 打磨**三个维度的全面升级。

### 核心问题

| # | 问题 | 严重度 |
|---|------|--------|
| 1 | `selectedMood` 情绪选择**从未传递到后端**，用户选什么情绪生成结果都一样 | 🔴 致命 |
| 2 | System Prompt 太粗糙（5 行指令），生成内容包含 `##` markdown、emoji、自我介绍 | 🟠 严重 |
| 3 | TTS 合成 252 字文案不分片，整段等待 ~10s 才能听 | 🟡 体验差 |
| 4 | `streamContent.length` 缺少 `.value`，toast 显示 `undefined字` | 🟡 Bug |

## 三、情绪分层 Prompt 体系

### 设计理念

每种情绪不只是"语气不同"，而是**节奏、意象、句式、情感弧线**全部不同。

### 基座 Prompt（DB `creative_template.ai_role`）

定义了情感文案大师的**通用写作能力**：

| 维度 | 规则 |
|------|------|
| TTS 铁律 | 纯净文本、无 markdown/emoji、标点控制呼吸、600 字上限 |
| 节奏韵律 | 短句为主、呼-呼-呼-吸韵律、长短交替 |
| 情感层次 | 画面开篇→共情→击中→温暖收束 |
| 文字质感 | 第二人称"你"、感官细节、口语化 |

### 情绪修饰符（前端 `mood.promptHint`）

6 种情绪各自携带完整的创作指令：

| 情绪 | 节奏设计 | 核心意象 | 情感弧线 | 金句方向 |
|------|----------|----------|----------|----------|
| 🌸 温柔治愈 | 溪水缓缓，≤15字/句 | 月光/微风/毛毯/热可可 | 画面→共情疲惫→认可→拥抱 | "你已经很努力了" |
| 🌧️ 感伤回忆 | 缓慢沉重，破折号留白 | 旧照片/空车站/褪色信 | 回忆→展开→轻抹痛处→释怀 | "有些人走了就是走了" |
| 🔥 热血励志 | 由慢到快，心跳加速 | 黎明/火焰/奔跑/破茧 | 共情低谷→转折→递进→一拳击胸 | "咬牙的日子终会发光" |
| 🌿 安静陪伴 | 极慢，不说话也不尴尬 | 台灯/茶杯热气/虫鸣 | "嘿"→安静→闲话→"在" | "不用说话，我在这" |
| 🌃 深夜独白 | 自言自语，断断续续 | 天花板/手机光/空酒杯 | 睡不着→独白→审问→原谅自己 | "明天的我会替今晚的我撑住" |
| ☀️ 早安能量 | 轻快明亮，拉开窗帘 | 阳光/咖啡/鸟鸣/露水 | 清晨画面→期待→挑战→出发 | "新的一天值得一个深呼吸" |

### Prompt 组装流程

```
最终 UserMessage = mood.promptHint + 600字硬限 + 用户灵感

示例：
【情绪：温柔治愈】
写作指引：
- 节奏：像溪水一样缓缓流淌，每句不超过15字
- 意象：月光、微风、棉花糖、暖阳...
- 语气：像在耳边低语，像拥抱...
- 结构：画面→疲惫→认可→拥抱
- 金句方向："你已经很努力了"...
【硬性要求】全文不得超过600字，这是铁律。

给深夜失眠的你，写一段温暖的话
```

## 四、TTS 合成加速策略

### 问题

情感电台文案 200-600 字，原 `splitTextForTTS(text, 280)` 阈值下，大部分不分片 → 整段同步等待。

### 方案：缩小分片 + 首片即播

```javascript
// 情感电台专用：100 字分片（其他类型保持 280）
const maxLen = project.value?.typeCode === 'radio' ? 100 : 280
const chunks = splitTextForTTS(section.content, maxLen)
```

| 文案长度 | 旧方案（280字分片） | 新方案（100字分片） |
|----------|---------------------|---------------------|
| 252 字 | 1 片，等 ~10s | 3 片，首片 ~3s 即播 |
| 450 字 | 2 片，等 ~6s | 5 片，首片 ~2s 即播 |
| 600 字 | 3 片，等 ~5s | 6 片，首片 ~2s 即播 |

首片合成完成 → `playerStore.play()` 立即播放
后续片段 → `playerStore.enqueue()` 入播放队列
全部完成 → `studioApi.concatAudio()` 拼接为单文件 → 持久化

## 五、前端安全过滤

即使 Prompt 明确禁止 markdown，LLM 仍可能偶尔输出 `##` 标题。增加前端实时过滤：

```javascript
// SSE 流处理中
if (project.value?.typeCode === 'radio') {
  text = text.replace(/^#{1,3}\s*/gm, '')           // 去 ## 标题
             .replace(/\*{1,2}([^*]+)\*{1,2}/g, '$1') // 去 **粗体**/*斜体*
}
```
