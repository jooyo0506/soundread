## 🚀 新特性

### AI 工作坊草稿自动保存

> 解决用户对话后刷新页面内容丢失的问题

| 文件 | 改动 |
|------|------|
| `DraftService.java` | **新增** 草稿 CRUD 服务，复用 `user_creation` 表（`type='agent_draft'`），最多保留 30 条，超限 LRU 淘汰 |
| `AgentController.java` | SSE `onComplete` 回调结束后非阻塞调用 `saveAgentDraft()`；新增 `GET /api/agent/drafts` 和 `DELETE /api/agent/drafts/{id}` 接口 |
| `AiWorkshop.vue` | 头部新增历史草稿按钮（🕐），底部抽屉展示所有草稿（文字+音频+时间+删除），零新增表结构 |

**设计要点**：
- 复用已有 `user_creation` 表，通过 `type` 字段区分草稿与普通 TTS 创作，无需 DDL 变更
- `extractAudioUrl()` 正则提取音频 URL，支持多种格式（`音频地址:`, markdown link, `.mp3`/`.wav`）
- 草稿保存在 SSE 全量回复确认后执行，保证内容完整性

## 🛡️ 缺陷修复

### Spring Boot 3.2 无映射资源 404 日志刷屏

| 文件 | 改动 |
|------|------|
| `GlobalExceptionHandler.java` | 新增 `@ExceptionHandler({NoResourceFoundException, NoHandlerFoundException})` 拦截器，从 `log.error` + 全量堆栈 → `log.warn` + 一行简短提示 |
| `ResultCode.java` | 新增 `NOT_FOUND(404, "接口不存在")` 枚举值 |

**背景**：Spring Boot 3.2 将找不到资源的请求改为抛出 `NoResourceFoundException`，原兜底的 `Exception.class` 处理器会把 404 记录为严重错误，导致日志刷屏（如 `api/upwload` 拼写错误）。

## ⚙️ 运维调整

### 暂停两个定时任务

| 任务 | 原频率 | 原因 |
|------|--------|------|
| `MusicService.pollMurekaTasks()` | 每 5 秒 | 当前音乐功能流量低，全量轮询 DB 开销不必要 |
| `HeatScoreJob.refreshAll()` | 每 5 分钟 | 发现页流量低，Redis ZSet 热度榜无需高频刷新 |

注释掉 `@Scheduled` 注解、保留方法体，如需恢复只需取消注释。
