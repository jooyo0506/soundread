## Bug 1：Agent Tool Calling 无限循环

### 现象

调用 `/api/agent/chat` 接口后，LLM 反复调用同一个 `@Tool` 方法（如 `listVoices`），导致后端死循环、最终超时 500。

### 根因

| 层面 | 原因 |
|------|------|
| **模型层** | Kimi-K2.5 (qwen) 对 Function Calling 协议的 **终止信号** 处理不好，收到工具结果后不生成文字回复，而是再次发出 `tool_calls` 请求 |
| **工具层** | `@Tool` 方法的返回值缺少"终结性"描述，LLM 不确定任务是否完成 |
| **系统指令层** | `@SystemMessage` 没有明确禁止重复调用 |
| **超时层** | `OpenAiChatModel` 未设置 `timeout`，循环时请求无限挂起 |

### 修复（四层防御）

| 层 | 修复内容 | 文件 |
|----|---------|------|
| **模型优先级** | Tool Calling 优先使用 MiniMax → DeepSeek（对 Function Calling 协议支持更好），Kimi-K2.5 降为兜底 | `AgentController.java` |
| **SystemMessage** | 加入"每个工具最多调一次，结果返回后直接总结回复" | `SmartAssistant.java` |
| **工具返回值** | 每个 `@Tool` 方法返回末尾加"（查询完毕，无需再次调用此工具）" | `SoundReadTools.java` |
| **超时保护** | `OpenAiChatModel.builder().timeout(Duration.ofSeconds(30)).maxTokens(1024)` | `AgentController.java` |

### 关键代码

```java
// SmartAssistant.java — 加了规则 3、4、5
@SystemMessage("""
    严格规则：
    3. 每个工具最多只调用一次，绝对不要重复调用同一个工具
    4. 工具返回结果后，直接用自然语言总结回复用户，不要再次调用工具
    5. 回答要简洁，不要重复输出工具返回的内容
    """)
```

```java
// AgentController.java — 超时 + 限制输出
toolModel = OpenAiChatModel.builder()
    .timeout(Duration.ofSeconds(30))
    .maxTokens(1024)
    .build();
```

### 教训

> **不是所有模型都擅长 Tool Calling。** 通用模型之间对 Function Calling 协议的实现差异很大。选型时必须实测"工具调用后能否正确终止"，不能只看速度和价格。

## Bug 2：登录后返回会重新进入登录页

### 现象

1. 用户在 Music 页 → 未登录 → 跳转到 `/login?redirect=/music`
2. 输入账号密码登录 → 回到 `/music`
3. 点击左上角返回 `←` → 浏览器回退到 `/login?redirect=/music`（又要登录）

### 根因

```
用户访问 /music → (未登录) router.push → /login → (登录成功) router.push → /music
                        ↑ push 留在历史栈        ↑ push 再加一层

浏览器历史栈：[/music] → [/login?redirect=/music] → [/music]
                              ↑ 点返回就回到这里！
```

核心问题：**所有跳转到登录页和从登录页跳走都用了 `router.push`**，导致登录页留在浏览器历史栈中。

### 修复

**统一原则**：所有涉及登录页的导航都改为 `router.replace`，登录页**永远不进入历史栈**。

```
修复后：
用户访问 /music → (未登录) router.replace → /login → (登录成功) router.replace → /music

浏览器历史栈：[/music]  ← 干净！点返回回到上一个正常页面
```

### 修改文件清单（共 9 个文件 · 13 处）

| 文件 | 改动 | 场景 |
|------|------|------|
| `Login.vue` | `push(redirect)` → `replace(redirect)` | 登录成功后跳转 |
| `Login.vue` | `router.back()` → `router.replace('/')` | 登录页返回按钮 |
| `request.js` | `push → replace` | 401 拦截器跳登录 |
| `Create.vue` | 2 处 `push → replace` | 生成台本/合成前鉴权 |
| `Emotion.vue` | 2 处 `push → replace` | AI 润色/合成前鉴权 |
| `Podcast.vue` | 1 处 `push → replace` | 生成播客前鉴权 |
| `Discover.vue` | 1 处 `push → replace` | 点赞前鉴权 |
| `Vip.vue` | 1 处 `push → replace` | 支付前鉴权 |
| `Profile.vue` | 2 处 `push → replace` | 登录按钮 + 退出登录 |
| `VoiceSelector.vue` | 1 处 `push → replace` | 购买音色前鉴权 |

### 关键知识点

```javascript
// push — 添加新历史记录（登录页留在栈里）
router.push('/login')    // history: [..., /music, /login]

// replace — 替换当前记录（登录页不留痕迹）
router.replace('/login') // history: [..., /login]  ← /music 被替换
```

> **SPA 导航铁律**：凡是"中间页"（登录页、授权页、支付回调页），一律用 `router.replace`，不要让它们污染浏览器历史栈。

## Bug 3：Chrome 控制台粘贴保护

### 现象

在 Chrome DevTools Console 粘贴代码时，提示：

> "Don't paste code into the DevTools Console that you don't understand..."

### 解决

在控制台手动输入 `allow pasting` 回车，即可解锁粘贴功能。
