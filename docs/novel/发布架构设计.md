# 统一发布架构设计

> 更新日期：2026-03-01

## 问题

当前灵感页（Discover）只展示 `work` 表中的**音频作品**。
后续需要支持多种内容类型发布：小说（文本）、播客（音频）、AI音乐（音频）。
需要一个统一架构，而不是每种类型单独造一套。

## 现有数据流

```
user_creation 表（创作记录）        work 表（发布到社区）
┌──────────────────────┐           ┌─────────────────────────┐
│ id, userId, type     │   发布    │ id, authorId, type      │
│ title, audioUrl      │ ───────→  │ title, audioUrl         │
│ isPublished, workId  │           │ coverUrl, description   │
│ extraJson            │           │ viewCount, likeCount    │
└──────────────────────┘           └─────────────────────────┘
        ↑                                    ↑
   TTS/情感/播客 合成后自动写入         发布到灵感页后才写入
```

**问题所在**：小说完结发布只更新了 `studio_project.status`，没有走这条链路。

## 推荐方案：统一 work 表 + 多类型渲染

### 核心思路

**`work` 表 = 灵感页的唯一内容源**。所有类型的内容发布，最终都是往 `work` 表插入一条记录。灵感页根据 `type` 字段差异化渲染卡片。

### work 表扩展

```sql
ALTER TABLE work ADD COLUMN content_type VARCHAR(32) DEFAULT 'audio' COMMENT '内容类型: audio/novel/podcast/music';
ALTER TABLE work ADD COLUMN word_count INT DEFAULT 0 COMMENT '字数（小说用）';
ALTER TABLE work ADD COLUMN chapter_count INT DEFAULT 0 COMMENT '章节数（小说用）';
ALTER TABLE work ADD COLUMN source_project_id BIGINT COMMENT '关联的 studio_project.id';
ALTER TABLE work ADD COLUMN extra_json TEXT COMMENT '扩展字段JSON';
```

### 各内容类型发布映射

| 字段 | 🎵 音频(TTS/情感) | 📖 小说 | 🎙️ 播客 | 🎶 AI音乐 |
|------|-------------------|---------|---------|-----------|
| `content_type` | `audio` | `novel` | `podcast` | `music` |
| `title` | 文本前20字 | 项目标题 | 播客标题 | prompt前20字 |
| `audioUrl` | ✅ R2链接 | ❌ null | ✅ R2链接 | ✅ R2链接 |
| `coverUrl` | 默认封面 | AI生成封面 | 播客封面 | 音乐封面 |
| `description` | 输入文本 | 故事梗概(synopsis) | 播客简介 | 风格描述 |
| `word_count` | 0 | 总字数 | 0 | 0 |
| `chapter_count` | 0 | 章节数 | 0 | 0 |
| `source_project_id` | null | project.id | project.id | task.id |
| `extra_json` | 音色/情感参数 | 角色表/大纲 | 嘉宾/话题 | 模型/参数 |

### 后端发布接口（统一入口）

```java
// StudioController.java — 新增发布端点
@PostMapping("/projects/{id}/publish")
public Result<Long> publishProject(@PathVariable Long id) {
    StudioProject project = studioService.getProject(id);
    
    // 1. 更新项目状态
    project.setStatus("completed");
    studioService.updateProject(project);
    
    // 2. 创建 Work 记录
    Work work = new Work();
    work.setAuthorId(project.getUserId());
    work.setTitle(project.getTitle());
    work.setContentType(project.getTypeCode()); // novel / podcast / ...
    
    if ("novel".equals(project.getTypeCode())) {
        work.setDescription(project.getOutline());  // synopsis 作为描述
        work.setWordCount(studioService.getTotalWordCount(id));
        work.setChapterCount(project.getTotalSections());
    }
    
    work.setSourceProjectId(project.getId());
    workService.save(work);
    
    return Result.ok(work.getId());
}
```

## 实施路径

### Phase 1（当前）— 后端可用后立即执行
1. ✅ `StudioController.updateProject` 支持 status 字段（已完成）
2. 新增 `work` 表的扩展字段 (`content_type`, `word_count`, `chapter_count`, `source_project_id`)
3. 新增 `/projects/{id}/publish` 端点
4. 前端 `publishNovel` 调用新端点
5. 灵感页适配小说卡片

### Phase 2 — 播客发布
1. 播客完成后调用同一个 `/projects/{id}/publish`
2. Work 自动填充 audioUrl、duration
3. 灵感页已支持音频播放，无需改动

### Phase 3 — AI 音乐发布
1. `MusicTask` 完成后创建 work（通过 `MusicService` 或独立 publish 接口）
2. content_type = 'music'
3. 灵感页复用音频播放逻辑

## 技术要点

### 为什么用 work 表而不是 user_creation？

| | work 表 | user_creation 表 |
|---|---------|-----------------|
| **定位** | 社区公开内容 | 个人创作历史 |
| **可见性** | 所有用户可见 | 仅创作者可见 |
| **社交属性** | viewCount, likeCount | 无 |
| **适合做灵感页** | ✅ | ❌ |

`user_creation` 是**个人资产**，`work` 是**社区内容**。
发布 = 从个人资产推送到社区。

### 小说阅读器页面

小说不是音频，点击后不能播放。需要新增一个独立的**小说阅读页** `/novel/:workId`，复用全屏阅读器组件。

```
路由: /novel/:workId
数据: 通过 workId → source_project_id → sections
展示: 全屏沉浸阅读器（已有组件）
```
