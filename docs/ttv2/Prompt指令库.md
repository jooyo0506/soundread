# 音色情感调音台 V2.0 —— 动态 AI 场景指令库 (Prompt Library)

> **版本定位**：记录本模块从原有的“前端硬编码 (Hardcode)”到“数据库支撑的 B 端全量动态化配置”的设计全景，作为后续维护与拓展的唯一指南。

---

## 🧭 一、架构蓝图与演进

### 1. 痛点与重构初衷
在早期版本中，大模型配音扮演的“角色指令卡片”（如：悬疑解说、温柔安抚师、激情带货）被写死在 Vue 前端字典组件中。这导致每次产品运营想要增加一个新的时下热门 IP（如“黑神话解说风”）时，都必须经过前端改码、打包、全量发版的漫长流程。这是典型的反 SaaS 模式。

### 2. 解决方案：数据下沉驱动
本次重构采用了 **DB 表驱动 + 后端 RESTful 嵌套 API + 前端挂载拉取** 的全栈闭环架构：
- **运营配置即时生效**：只要在数据库（或未来的运营后台）中插入一行数据，全网所有 C 端用户的界面将立刻多出一个精美的角色卡片。
- **前后端彻底解耦**：前端彻底剥离了沉重的业务定义，蜕变为一个纯粹的“渲染引擎”。

---

## 🗄️ 二、数据库表结构设计 (Schema)

为保证场景分类的灵活性，系统采用了经典的 **大类 (Category) -> 角色 (Role)** 一对多层级结构设计：

### 1. 分类表 (`ai_prompt_category`)
承载如“广播剧”、“短视频”、“教育助手”等宏大分类。
- `id` (PK)
- `name`：分类名称
- `icon`：分类图标的 Font Awesome Class (如 `fas fa-headphones-alt`)
- `sort_order`：控制页面展示顺序

### 2. 角色设定表 (`ai_prompt_role`)
承载具体的人物画像标签与 Prompt 指令设定。
- `id` (PK)
- `category_id`：所属大类 FK
- `name`：角色短名（如 “悬疑惊悚” / “霸道总裁”） 
- `description`：前端 UI 展示的提示词（如 “压低声音、带点气声、放缓、偶尔颤音”）
- `tags`：实际拼装发给大模型解析引擎的精细化 Tag 标签串
- `sort_order`：卡片在类别中的展示顺序

## 🎭 四、大模型神经链路穿透 (Context Injection)

本次重构不止于 UI 动态化，更深入到了 **AI Prompt Engineering** 骨髓：

当用户在界面中点击某张动态拉取的角色卡片（例如“激情带货”）时：
1. **指令拼装**：提取 `tags`，自动填充文本框为 `[#用语速极快、充满煽动性、语调高昂的语气]`。
2. **隐藏变量注入**：提取该卡片的 `name`（“激情带货”），赋值给 Vue 的 `sceneTheme`。
3. **同步提交网关**：`AiSceneRequest` 会携带 `instruction`、`theme` 以及新增的 **`context` (剧情上文)** 三重参数发往 Java 中枢。
4. **Prompt 重塑与语境增强**：
   在 `AiScriptService` 中，System Prompt 骨架由原先的单轨升维到了**三维一体注入**：
   > “你的任务是根据用户提供的：
   > 1. 【全局语气指令】(*instruction参数*) 
   > 2. 【前情提要/背景】(*context参数*) 
   > 3. 【角色/场景主题】(*theme参数*)
   > 用丰富的笔触扩写一段专门用于【单人旁白】的配音稿...”

这套机制直接填补了大模型在生成文本时的**情境空白**，让它不仅知道“用什么语气说”，更知道了自己“此刻是以什么身份身处什么片场，且刚发生了什么”，大幅拉升了台本生成的商业爆发力。

## 📝 五、运营更新指南

未来按本模式更新指令库步骤极其简单：
1. 确定新的业务线想要什么场景/角色。
2. 直连 MySQL，向 `ai_prompt_category` 中 `INSERT` 大类（若需要）。
3. 向 `ai_prompt_role` 表中 `INSERT` 具体角色信息，精心设置 `tags`（如“清冷、高智商感”）。
4. 确认 `sort_order`（越小越排前）。
5. 用户刷新页面即刻生效。
