# 火山引擎双向流式TTS API 整理文档

## 1. 概述
双向流式API提供文本转语音能力，支持多语种、多方言，基于WebSocket协议实现流式交互。可一次性发送完整文本，也支持边发送文本边接收音频的流式模式。

**核心优势**：  
- 自动处理碎片化或超长文本，平衡延迟与合成效果。  
- 推荐将大模型流式输出直接输入接口，无需额外切句或攒句，合成更自然。  
- 支持连接复用：同一WebSocket连接内可多次发起会话（session），提升效率。

## 2. 快速开始

### 2.1 前提条件
- **APP ID**：从火山引擎控制台获取（[查看指引](https://www.volcengine.com/docs/6561/196768#q1)）
- **Access Token**：从火山引擎控制台获取
- **音色ID**：选择所需音色（[音色列表](https://www.volcengine.com/docs/6561/1257544)）

### 2.2 WebSocket建连与鉴权
- **请求路径**：`wss://openspeech.bytedance.com/api/v3/tts/bidirection`
- **请求头（HTTP Upgrade时携带）**：

| Header | 说明 | 必选 | 示例 |
|--------|------|------|------|
| `X-Api-App-Key` | APP ID | 是 | `123456789` |
| `X-Api-Access-Key` | Access Token | 是 | `your-access-key` |
| `X-Api-Resource-Id` | 资源ID（模型类型） | 是 | `seed-tts-1.0`、`seed-tts-2.0` 等 |
| `X-Api-Connect-Id` | 连接追踪ID（建议唯一） | 否 | `67ee89ba-...` |
| `X-Control-Require-Usage-Tokens-Return` | 返回用量标记 | 否 | `*` 或 `text_words` |

- **响应头**：握手成功后返回 `X-Tt-Logid`，用于问题定位。

### 2.3 基础交互流程
1. **建立连接**：发送 `StartConnection` 事件，收到 `ConnectionStarted` 后连接建立。
2. **开启会话**：发送 `StartSession`（携带合成参数），收到 `SessionStarted`。
3. **发送文本**：通过 `TaskRequest` 事件发送文本，同时接收音频数据。
4. **结束会话**：文本发送完毕，发送 `FinishSession`，收到 `SessionFinished`。
5. **复用连接**：如需继续合成，重复步骤2~4；否则发送 `FinishConnection` 关闭连接。

> ⚠️ 同一连接下不支持多个session并行，必须串行。

## 3. API详细说明

### 3.1 WebSocket二进制协议
消息帧结构（所有整数均为**大端**）：

| 字节偏移 | 内容 | 说明 |
|----------|------|------|
| 0 (左4位) | Protocol version | 目前固定 `0b0001` (v1) |
| 0 (右4位) | Header size | 目前固定 `0b0001` (4字节) |
| 1 | Message type + flags | 详见下表 |
| 2 (左4位) | Serialization | `0b0000`=raw, `0b0001`=JSON |
| 2 (右4位) | Compression | `0b0000`=无压缩, `0b0001`=gzip |
| 3 | Reserved | 留空 |
| 4~7 | [Optional event number] | 仅当flags包含event时存在 |
| ... | Payload | 音频数据或JSON文本 |

**Message type & flags**：

| Message type | 含义 | flags | 是否含event | 备注 |
|--------------|------|-------|-------------|------|
| 0b0001 | Full-client request | 0b0100 | 是 | 上行请求（含event） |
| 0b1001 | Full-server response | 0b0100 | 是 | 下行响应（含event） |
| 0b1011 | Audio-only response | 0b0100 | 是 | 纯音频下行 |
| 0b1111 | Error information | 无 | 否 | 错误帧 |

### 3.2 请求参数（StartSession 时设置）
| 字段 | 说明 | 必选 | 类型 | 默认 |
|------|------|------|------|------|
| `user.uid` | 用户ID | 否 | string | - |
| `event` | 事件类型（固定100） | 是 | int | - |
| `req_params.text` | 合成文本 | 是 | string | - |
| `req_params.speaker` | 音色ID | 是 | string | - |
| `req_params.model` | 模型版本（如 `seed-tts-1.1`） | 否 | string | - |
| `req_params.audio_params.format` | 音频格式：mp3/ogg_opus/pcm | 否 | string | mp3 |
| `req_params.audio_params.sample_rate` | 采样率：8000~48000 | 否 | number | 24000 |
| `req_params.audio_params.bit_rate` | 比特率（建议主动设置） | 否 | number | - |
| `req_params.audio_params.emotion` | 情感（部分音色支持） | 否 | string | - |
| `req_params.audio_params.speech_rate` | 语速（-50~100） | 否 | number | 0 |
| `req_params.audio_params.loudness_rate` | 音量（-50~100） | 否 | number | 0 |
| `req_params.audio_params.enable_timestamp` | TTS1.0字级时间戳 | 否 | bool | false |
| `req_params.audio_params.enable_subtitle` | TTS2.0字幕 | 否 | bool | false |
| `req_params.additions` | 扩展参数（JSON字符串） | 否 | string | - |

**常用扩展参数（additions）**：
- `silence_duration`：句尾增加静音时长(0~30000ms)
- `disable_markdown_filter`：是否过滤Markdown语法
- `disable_emoji_filter`：是否保留Emoji
- `explicit_language`：指定语种（如 `zh-cn`, `en`）
- `cache_config`：开启缓存（相同文本加速）
- `context_texts`：辅助情感/语速调整（TTS2.0）
- `use_tag_parser`：开启COT标签解析（ICL2.0）

> 更多参数详见原文档，此处仅列核心。

### 3.3 响应与事件定义
下行事件通过 `event` 字段区分：

| Event code | 含义 | 方向 | 说明 |
|------------|------|------|------|
| 1 | StartConnection | 上行 | 发起连接 |
| 2 | FinishConnection | 上行 | 关闭连接 |
| 50 | ConnectionStarted | 下行 | 连接成功 |
| 51 | ConnectionFailed | 下行 | 连接失败 |
| 52 | ConnectionFinished | 下行 | 连接已关闭 |
| 100 | StartSession | 上行 | 发起会话 |
| 101 | CancelSession | 上行 | 取消会话 |
| 102 | FinishSession | 上行 | 结束会话 |
| 150 | SessionStarted | 下行 | 会话开始 |
| 151 | SessionCanceled | 下行 | 会话已取消 |
| 152 | SessionFinished | 下行 | 会话结束（含用量） |
| 153 | SessionFailed | 下行 | 会话失败 |
| 200 | TaskRequest | 上行 | 发送文本 |
| 350 | TTSSentenceStart | 下行 | 句子开始 |
| 351 | TTSSentenceEnd | 下行 | 句子结束（含时间戳） |
| 352 | TTSResponse | 下行 | 音频数据 |

**TTS1.0与TTS2.0的时间戳差异**：
- TTS1.0：每个子句独立返回 `TTSSentenceStart/End`，字幕跟随 `End` 返回，基于TN打轴。
- TTS2.0：整个会话只返回一次 `Start/End`，字幕通过 `TTSSubtitle` 事件多次返回，基于原文打轴。

## 4. 交互示例图解

### 4.1 TTS1.0 / ICL1.0 交互
```
Client                     Server
  |----StartConnection----->|
  |<--ConnectionStarted-----|
  |----StartSession-------->|
  |<--SessionStarted--------|
  |----TaskRequest(text1)-->|
  |<--TTSSentenceStart------|
  |<--TTSResponse(audio1)---|
  |<--TTSSentenceEnd--------| (含时间戳)
  |----TaskRequest(text2)-->|
  |<--TTSSentenceStart------|
  |<--TTSResponse(audio2)---|
  |<--TTSSentenceEnd--------|
  |----FinishSession------->|
  |<--SessionFinished-------|
  |----FinishConnection---->|
  |<--ConnectionFinished----|
```

### 4.2 TTS2.0 / ICL2.0 交互
```
Client                     Server
  |----StartConnection----->|
  |<--ConnectionStarted-----|
  |----StartSession-------->|
  |<--SessionStarted--------|
  |----TaskRequest(text)---->|
  |<--TTSSentenceStart------|
  |<--TTSResponse(audio1)---|
  |<--TTSSubtitle(sub1)-----| (可能延迟)
  |<--TTSResponse(audio2)---|
  |<--TTSSubtitle(sub2)-----|
  |<--TTSSentenceEnd--------|
  |----FinishSession------->|
  |<--SessionFinished-------|
  |----FinishConnection---->|
  |<--ConnectionFinished----|
```

> `CancelSession` 可在 `SessionStarted` 后、`FinishSession` 前发送，用于主动放弃当前合成。

## 5. 错误码
| 错误码 | 含义 |
|--------|------|
| 20000000 | 成功 |
| 45000000 | 客户端通用错误 |
| 45000001 | 请求参数错误 |
| 55000000 | 服务端通用错误 |
| 55000001 | 服务端session错误 |

错误响应帧中会包含 `status_code` 和 `message`。

## 7. 注意事项
- 同一个WebSocket连接不支持并发session。
- 建议复用连接，避免频繁建连开销。
- 音频格式若选择 `wav`，在流式场景下会多次返回wav header，建议改用 `pcm`。
- 开启缓存可加速相同文本合成，但不会附带时间戳。
- 情感、语速等调整参数需确认音色支持范围。
