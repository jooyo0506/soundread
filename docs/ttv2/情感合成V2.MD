# 语音合成高级控制功能及应用场景

## 一、功能概述

本功能通过三种高级控制方式，显著增强语音合成的表现力和情感丰富度，使生成的语音更自然、更具感染力，有效解决传统 TTS 情感单一、缺乏语境理解的问题。

- **语音指令**  
  在合成文本前添加特定指令（如 `[#用asmr的语气]`），可精准控制整体情绪、方言、语气、语速、音调等，实现从“吵架”到“悄悄话”等多种风格切换。

- **引用上文**  
  提供一段对话历史作为上下文，模型能理解并承接其中的语境与情感，使合成语音更贴合真实对话场景（如表现出思考、停顿、激动等）。


## 二、前后端技术实现规约 (2026 最新整合)

为了降低普通用户的输入心智负担，我们在前端保留了最自然、最易懂的书写习惯（例如 `[#]` 与 `【】`），而在后端实现了基于正则表达式的**指令编译器**，将其无缝转换为火山引擎底层的繁琐 XML 与 JSON 协议：

1. **语法糖解析机制**
   - **全局指令转换**：当用户在界面中输入 `[#用asmr的语气]` 这种自然语言标签时，后端会将其正则表达式剥离，并推入火山引擎 `req_params.additions.context_texts` 的字符串数组中，从而激活其隐藏的语气指令槽。
   - **局部情绪转译（COT）**：当用户使用诸如 `【哭腔】`、`【叹气】` 这类局部控制时，后端引擎自动将其打包为官方的变种 XML 格式 `<cot text=表情>...</cot>` 中。
   - **打通参数层**：并在发送到大模型 WebSocket 节点时，强制在 payload 注入附加参数 `"use_tag_parser": true` 以挂载原生模型解析器。

2. **前端沉浸式专属座舱 UI**
   - 不再复用传统的 `Create.vue` 界面，专门重构了 `Emotion.vue` 作为“调音台”。
   - 提供直接的快捷模板面板支持（包含播音腔、鬼故事等快速 Prompt 注入）。
   - 实装上文情境 (`Context`) 面板，彻底打通剧情与对话推演逻辑。

## 三、核心应用场景

这些功能主要应用于需要声音具备**情感表现力**、**角色代入感**或**特定氛围**的内容创作领域。

### 1. 有声内容创作
- **有声书/广播剧**  
  - **多角色演绎**：通过语音标签区分旁白、不同角色对话。  
    - 例：`【微笑】夜幕降临...` 和 `【愤怒】你给我站住！`  
  - **情感渲染**：利用语音指令快速切换角色情绪（如悲伤独白→激烈呼喊），或表现人物内心矛盾。

- **播客/故事讲述**  
  用“引用上文”模拟真实对话场景，让讲述更有对象感和交流感，避免平铺直叙。

### 2. 影视与游戏配音
- **游戏角色配音**  
  为 NPC 或主角赋予独特性格与情感。  
  - 傲娇公主：`[#用撒娇又带点命令的语气]`  
  - 沧桑老兵：`[#用低沉沙哑、带着疲惫的语气]`

### 3. 社交媒体与创意内容
- **短视频配音/创意广告**  
  快速生成贴合角色与剧情的配音，尤其适合方言（如四川话）或特定语气（如夹子音、激情促销等）。

## 总结
本技术的核心价值在于**打破传统 TTS 的机械感和情感单一局限**，让创作者能够像使用画笔一样精细调教出**充满情绪、符合语境、独具个性**的声音，在各种内容载体上创造更具感染力的听觉体验。
